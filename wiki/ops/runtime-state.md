# Runtime State

Runtime and sensitive files explain how this Codex install operates locally, but they are not durable source material for Git or wiki ingest.

## Sensitive Files

`auth.json`, `.credentials.json`, and `installation_id` are local identity or credential files. Do not read, summarize, quote, or commit their contents.

## Databases And History

SQLite files and their WAL/SHM companions are local Codex databases. `history.jsonl`, `sessions/`, and `shell_snapshots/` are conversation and terminal history. These may contain private prompts, outputs, paths, or tool payloads; do not ingest them.

## Runtime Directories

`cache/`, `tmp/`, `.tmp/`, `log/`, `app-server-control/`, and `memories/` are local runtime or generated state. They can be deleted or regenerated by Codex behavior and should stay outside the long-term wiki.

## Standalone Package

`packages/standalone/` contains Codex standalone release binaries. The current symlink points at release `0.142.5-x86_64-unknown-linux-musl`. Only the version and role should be recorded; binary contents are excluded from ingest.

## Other Ignored Files

`.gitignore` also excludes `models_cache.json`, `version.json`, `.personality_migration`, Python bytecode caches, `backups/`, and `*.bak-*`. Backup files can be inspected only after an explicit task asks to recover or compare them.

## Maintenance Rule

Before adding any new file to Git or wiki, classify it as durable configuration/source or local runtime state. Durable files should be documented in the relevant wiki page; runtime files should only appear here as categories.
